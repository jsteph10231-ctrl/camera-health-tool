## Design Context

### Users
Internal LSU-facing users working in a camera-health and operations context. They use the tool to monitor system status, review device health, process operational workflows, and act quickly on camera issues without extra UI friction.

### Brand Personality
Functional, useful, data-driven. The interface should feel dependable and operationally focused, with clarity and speed taking priority over decoration. It should support confidence and fast decision-making during routine monitoring and issue follow-up.

### Aesthetic Direction
The product should read primarily as an operations console. Dark mode should remain the default presentation. Existing LSU visual cues such as purple and gold can continue to appear, but they should support the operational feel rather than overpower it. Visual choices should stay practical, restrained, and information-first rather than flashy or ornamental.

### Design Principles
1. Prioritize function first: every screen should help users find status, understand risk, and take the next action quickly.
2. Make data easy to scan: emphasize hierarchy, contrast, labeling, and compact summaries over decorative UI.
3. Preserve the ops-console feel: use dark, focused, control-room styling as the default visual posture.
4. Use LSU branding with restraint: keep recognizable color cues, but avoid letting branding reduce readability or operational clarity.
5. Favor dependable interactions: reduce visual noise, avoid gimmicks, and make controls feel stable and predictable.
